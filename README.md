# SMSCenter

一个能够将 Android 手机短信实时同步到网页查看的项目，适合希望在电脑上快速查看验证码、通知短信等内容的场景。

---

# 目录结构

```text
SMSCenter/
├── SMSCenter/                 # Android 客户端
└── pc_side/
    └── Receiving_interface/   # FastAPI 接收服务
```

---

# 一、介绍

SMSCenter 是一个将 Android 手机短信同步到网页的项目。

手机客户端监听短信通知后，会通过 HTTPS 上传到服务器，随后即可在网页上实时查看短信内容，无需频繁查看手机。

---

# 二、启动方法

## Android 客户端

使用 Android Studio 打开 `SMSCenter` 项目，编译并安装到手机即可。

## 服务端

运行 `pc_side/Receiving_interface` 中的 FastAPI 服务即可接收短信。

---

# 三、手机设置

为了保证短信能够实时同步，请完成以下配置：

## 1. 开启权限

- 通知使用权（Notification Listener）
- 通知权限

## 2. 关闭电池优化

防止系统自动杀死后台进程。

## 3. 不同品牌手机设置

| 品牌 | 设置方法 |
|------|---------|
| **华为** | 手机管家 → 应用启动管理 → SMSCenter → 关闭"自动管理" → 三项全开（自启动 / 关联启动 / 后台活动） |
| **小米** | 安全中心 → 应用管理 → SMSCenter → 自启动：开启；省电策略：无限制 |
| **OPPO / 一加** | 设置 → 电池 → 更多 → SMSCenter → 允许后台运行 + 关闭自动冻结 |
| **vivo** | i管家 → 应用管理 → SMSCenter → 自启动 + 后台弹出界面 |
| **三星** | 设置 → 设备维护 → 电池 → 未监视的应用程序 → 添加 SMSCenter |
| **原生 Android / 类原生** | 一般无需额外设置 |

# 四、环境要求

- Android **9.0（API 28）** 及以上
- 推荐编译 SDK：**Android 15（API 36）**
- 推荐 Kotlin：**2.2.10**（不是强制要求）

---

# 五、注意事项

1. 通知栏常驻服务不可关闭，否则无法接收短信。
2. 必须授予通知监听权限。
3. 必须关闭系统的电池优化，否则后台可能会被系统终止。
4. 当前项目默认上传到开发者配置的服务器。如果直接使用源码而**不修改服务器地址**，你的短信将会上传到该服务器，并且你无法查看这些短信。因此，请在使用前将服务器地址修改为你自己的部署地址。
